= SLO de sostenibilidad para Keystone
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
El objetivo de nivel de servicio de sostenibilidad (SLO) apunta al consumo de energía hasta el máximo de vatios nominales para los productos STaaS implementados para ofrecer la instancia de nivel de servicio de rendimiento (PSLI) para almacenamiento unificado (excluido el nivel de servicio de rendimiento _Value_) y optimizado por bloques, o por clúster AFX. Conoce cómo el objetivo de sostenibilidad de Keystone afecta tu configuración STaaS y cómo obtener créditos de servicio si no se cumplen los límites.



== Métricas

* *Vatios*: La energía consumida por las matrices de almacenamiento, o clúster AFX, y según lo informado por la herramienta de supervisión. Los límites máximos de consumo de energía de referencia se proporcionan en link:https://hwu.netapp.com["NetApp Hardware Universe"^].
* *Tebibyte*: El máximo de:
+
** la capacidad comprometida + la capacidad de ráfaga asignada para el nivel de servicio de rendimiento, o
** la capacidad instalada efectiva, asumiendo un factor de eficiencia del almacenamiento de `2:1`.




Para obtener más información sobre la tasa de eficiencia del almacenamiento, consulte https://docs.netapp.com/us-en/active-iq/task_analyze_storage_efficiency.html["Analizar el ahorro en capacidad y la eficiencia del almacenamiento"^].



=== Configuraciones compatibles

El SLO de sostenibilidad se aplica a los pedidos estándar de Keystone STaaS con las siguientes configuraciones compatibles:

* Solo matrices de almacenamiento ONTAP AFF Serie A o clústeres AFX.
* Solo configuraciones de clúster sin switches de dos nodos.
* Sólo configuraciones no combinadas.
* Sólo configuraciones sin MetroCluster.
* Habilitación completa y continua de la herramienta de supervisión y AutoSupport (ASUP) en todas las matrices de almacenamiento.
* La temperatura ambiente de funcionamiento durante la medición del consumo de energía debe estar comprendida entre 25°C y 27°C.




== Créditos de servicios


NOTE: Los acuerdos de nivel de servicio y las garantías están disponibles según las nominaciones.

Si el consumo de energía durante un periodo de facturación supera los vatios nominales máximos para los productos STaaS desplegados, NetApp emite créditos de servicio basados en la capacidad comprometida de la siguiente manera:

|===
| Días de incumplimiento de SLA en el período de facturación | Crédito de servicio

|===


=== Solicitud de crédito de servicio

Para solicitar un crédito de servicio, abre un ticket de soporte de prioridad 3 (P3) con NetApp Keystone support dentro de los 7 días posteriores al final del mes de servicio y proporciona los detalles que se solicitan en la hoja de Excel como se muestra a continuación:

image:sla-breach.png["plantilla de hoja de excel"]

Ten en cuenta las siguientes condiciones para las solicitudes de crédito de servicio:

* NetApp determina si se ha producido un incumplimiento del nivel de servicio.
* Debes proporcionar información adicional que te solicite NetApp, incluidos tus datos de consumo de vatios y el método de medición.
* Los créditos de servicio se calculan únicamente sobre la capacidad comprometida.
* Los créditos de servicio sólo pueden aplicarse a la factura del mes siguiente. No se aplican a las suscripciones a Keystone caducadas. Para contactar con el soporte de NetApp y abrir el caso de soporte necesario para solicitar un crédito de servicio, consulta link:../concepts/support-services.html["Obtén ayuda con Keystone"].
